Notes for demo - showing data extraction from IDS (HL7 already parsed by Atos) and
insertion to UDS.

0. Check docker image configured properly
	- ifconfig - get en0 IP
	- cd docker; edit config.json
	- docker image build -t test .

1. Run script to reset entries in INFORM_SCRATCH (i.e. set to empty tables):
	psql -f create_tables.cmd INFORM_SCRATCH

2. Delete entries in DUMMY_IDS and add first two back in:
	psql -f create_dummy_ids.cmd DUMMY_IDS

3. Login to IDS and see only two records there:
	psql DUMMY_IDS
	select UNID, patientsurname, hospitalnumber, patientlocation, admissiondate, dischargedate from tbl_ids_master;

4. Log in to UDS and see empty:
	psql INFORM_SCRATCH
	select * from person;
	select visitid, hospitalnumber, patientlocation, admissiondate, dischargedate, lastupdated from patient_visit;

5. Run Java docker instance to extract info from IDS to UDS:
	 docker container run test

6. Show that UDS now has some data in it:
	select * from person;
        select visitid, hospitalnumber, patientlocation, admissiondate, dischargedate, lastupdated from patient_visit;

7. Add a patient admission to IDS:
	psql -f insert_record_to_dummy_ids.cmd DUMMY_IDS
	select UNID, patientsurname, hospitalnumber, patientlocation, admissiondate, dischargedate from tbl_ids_master;

8. Run docker again:
	docker container run test

9. Show data in UDS:
	select * from person;
	select visitid, hospitalnumber, patientlocation, admissiondate, dischargedate, lastupdated from patient_visit;

10. Discharge Leonardo da Vinci:
	psql -f insert_discharge_message.cmd DUMMY_IDS

11. Run docker container again to update UDS:
	docker container run test

12. Look at UDS to see changes:
	select visitid, hospitalnumber, patientlocation, admissiondate, dischargedate, lastupdated from patient_visit;
